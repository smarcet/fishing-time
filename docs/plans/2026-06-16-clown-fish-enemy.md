## Summary

**Goal:** Add a `ClownFish` enemy (extends `CatchableFish`) with strength=5 and escape_rate=1.2 assembled from the Fish_2 PNG frames, registered in EnemyFactory, spawned in Game, and documented in an ADR.

## Approach

**Chosen:** Follow the `LionFish`/`Tuna` `spriteFrameSize` pattern — canonical cell 342×321 px (die-frame size), 2-row sheet, `ClownFish` extends `CatchableFish` with separate `_sw`/`_sh`, `draw()` and `_drawCapturedSprite()` overrides.

**Why:** Fish_2 ships two animation groups with different natural sizes (move: 231×135, die: 342×321). Using the larger die-frame as canonical (as SwordFish/Tuna/LionFish do) avoids distorting the captured-pose sprite; the move frames are simply centred in the larger cell with a transparent surround. The `spriteFrameSize` pattern decouples source stride from display size without touching the base class.

## Context for Implementer

The Fish_2 asset pack has three animation groups; only `Fish_move_2` (10 frames, 231×135 px) and `Fish_die_2` (10 frames, 342×321 px) are used. The `Fish_attack_2` group is skipped, matching the precedent set by Tuna (unused animation groups stay unwired). During the spritesheet assembly, each move frame is centred inside a 342×321 transparent cell; die frames paste at full size. The assembled PNG must be placed at `images/fishes/clown_fish_sprite.png` before the game runs. All `ctx.scale(-1,1)` flip logic for rightward movement is inherited verbatim from LionFish.

### Task 5: Write ADR 0015

**Objective:** Document all ClownFish design decisions in `docs/adr/0015-clown-fish-enemy.md`, following the format and depth of ADR 0014 (Tuna).

**Files:**

- Create: `docs/adr/0015-clown-fish-enemy.md`

**Key Decisions / Notes:**

- Sections: Context, Decisions (one heading per decision), no Consequences section (project convention).
- Decisions to document (following ADR 0014 structure):
  1. `ClownFish` extends `CatchableFish` via `spriteFrameSize` pattern (same rationale as LionFish/Tuna)
  2. Spritesheet layout: 2-row 3420×642, canonical cell 342×321, move frames centred, die frames as-is
  3. Animation groups used vs. unused (`Fish_attack_2` skipped — same rationale as Tuna unused animations)

  5. Display size: 114×107 px (~1/3 of canonical), matching visual size of ButterflyFish tier
  6. Drift speed: 1.5 px/tick (same as ButterflyFish — consistent easy-tier speed)
  7. Spawn zone: full water column (same as ButterflyFish)
